# Sample Flow

## Prompt
"We need an architecture for a university attendance system that supports web and mobile check-in, staff dashboards, auditability, and future integrations with identity, timetabling, and notifications. The team is small and wants something maintainable first, but the system may expand across multiple campuses later."

## Suggested flow
1. architecture-option-generator
2. quality-attribute-scenario-writer
3. tradeoff-analysis-writer
4. monolith-vs-modular-monolith-reviewer
5. layered-architecture-designer
6. component-boundary-reviewer
7. integration-boundary-mapper
8. runtime-view-writer
9. deployment-view-writer
10. scalability-hotspot-detector
11. availability-strategy-reviewer
12. architecture-risk-assessor
13. adr-writer

## Expected artifacts
- architecture options table with rationale
- quality attribute scenarios with priority notes
- explicit tradeoff matrix and recommendation
- boundary map for major components and integrations
- runtime and deployment views
- hotspot and availability findings
- architecture risk register
- ADR capturing the chosen structure
